## 소개

PTZ Controller는 PelcoD, ONVIF, Ujin 등 다양한 프로토콜을 지원하는 웹 기반 PTZ 카메라 제어 시스템입니다. 웹 브라우저 또는 Windows 데스크톱 앱(Electron)으로 사용할 수 있습니다.

### 주요 기능

| 기능            | 설명                                                         |
| --------------- | ------------------------------------------------------------ |
| 다중 프로토콜   | PelcoD, ONVIF, Ujin(PelcoD 확장), 커스텀                     |
| PTZ 제어        | Pan, Tilt, Zoom, Focus, Iris                                 |
| 프리셋          | 최대 255개 저장/호출/삭제                                    |
| AUX 제어        | PelcoD 6채널, Ujin 2채널 On/Off                              |
| 카메라 설정     | AF, AI, AGC, BLC, AWB                                        |
| 자동 위치 쿼리  | 설정 주기(ms)로 카메라 위치 자동 폴링                        |
| ONVIF 자동 인증 | 제조사별 인증 방식 자동 협상                                 |
| Hex 모니터      | 실시간 TX/RX 패킷 16진수 표시                                |
| 테마            | Light, Dark, System (기본: System)                           |
| 비밀번호 재설정 | 관리자 강제 리셋 + 사용자 셀프서비스 (이메일)                |
| 관리자 기능     | 사용자 관리, Proxy 배포 파일 업로드, Cloud Download URL 설정 |

## 시스템 구성

### 4가지 구성 요소

| 구성 요소           | 폴더                     | 설명                                   |
| ------------------- | ------------------------ | -------------------------------------- |
| **Web App (Admin)** | `ptzcontroller_admin/`   | Next.js 웹 서버 (UI + API)             |
| **Desktop App**     | `ptzcontroller_desktop/` | Electron 래퍼 (Windows EXE)            |
| **PTZ Proxy**       | `ptz-proxy-electron/`    | WebSocket ↔ TCP 프록시 (GUI 트레이 앱) |
| **License Server**  | `ptz-license-server/`    | 라이선스 발급/관리 서버                |

### 전체 통신 구조

```
┌─────────────────────────────────────────────────────────────┐
│                        Internet                             │
│  ┌──────────────┐                   ┌────────────────────┐  │
│  │  Web Server  │◄──── HTTPS ───────│     Browser        │  │
│  │  (Cloud)     │                   │   (User's PC)      │  │
│  └──────────────┘                   └────────┬───────────┘  │
│                                              │ WebSocket    │
│                                     ┌────────▼───────────┐  │
│                                     │ ptz-proxy-electron │  │
│                                     │ (User's PC)        │  │
│                                     └────────┬───────────┘  │
└──────────────────────────────────────────────│──────────────┘
                                               │ TCP
                                ┌──────────────▼────────────┐
                                │     Local Network         │
                                │   ┌──────────────┐        │
                                │   │  PTZ Camera  │        │
                                │   │ 192.168.x.x  │        │
                                │   └──────────────┘        │
                                └───────────────────────────┘
```

### 웹 브라우저 요구사항

Chrome 90+, Firefox 88+, Safari 14+, Edge 90+

## 계정 및 로그인

### 첫 번째 가입자 자동 admin

**최초 회원가입 시 자동으로 admin(관리자) 권한이 부여됩니다.**
DB에 seed 계정(`john@doe.com`)이 있더라도 이 계정은 제외하고 판단합니다.

### 회원가입

1. 로그인 화면 하단의 **"Don't have an account? Sign up"** 을 클릭합니다.
2. 다음 항목을 입력합니다.

| 항목      | 필수 | 설명            |
| --------- | ---- | --------------- |
| 이름      | ✅   | 사용자 이름     |
| 회사/소속 | ✅   | 소속 기관       |
| 이메일    | ✅   | 로그인용 이메일 |
| 비밀번호  | ✅   | 8자 이상        |

3. **"Create Account"** 를 클릭하면 가입 완료 후 자동 로그인됩니다.

### 로그인

1. 이메일과 비밀번호를 입력합니다.
2. **"Sign In"** 을 클릭하면 대시보드로 이동합니다.

### Seed 테스트 계정 (개발/테스트용)

`yarn prisma db seed` 실행 시 자동 생성:

- Email: `john@doe.com`
- Password: `hellotyche!`

> ⚠️ 운영 환경에서는 이 계정의 비밀번호를 변경하거나 삭제하세요.

## 카메라 추가 및 편집

카메라 목록의 `+` 버튼 또는 ✏️ 버튼을 클릭하면 모달이 나타납니다.

### 기본 설정

| 필드                | 설명                          | 예시                        |
| ------------------- | ----------------------------- | --------------------------- |
| Camera Name         | 카메라 식별 이름              | 1번 카메라                  |
| Protocol            | 통신 프로토콜                 | PelcoD, Ujin, ONVIF, Custom |
| PTZ 카메라 IP       | 카메라 IP (Proxy 기준)        | 192.168.1.100               |
| Port                | 통신 포트                     | 4001                        |
| Device Address      | 장치 주소 1–255 (PelcoD/Ujin) | 1                           |
| Proxy WebSocket URL | PTZ Proxy 주소                | ws://localhost:9902         |

### ONVIF 전용 설정

| 필드          | 설명                               |
| ------------- | ---------------------------------- |
| Username      | ONVIF 사용자명                     |
| Password      | ONVIF 비밀번호                     |
| Profile Token | 프로파일 토큰 (비워두면 자동 조회) |

**ONVIF Profile Token 자동 스캔:**

1. Username과 Password를 입력합니다.
2. 스캔 버튼을 클릭하면 카메라에서 사용 가능한 프로파일 목록을 조회합니다.
3. 각 프로파일에 Name, Token ID, Resolution, PTZ 지원 여부가 표시됩니다.
4. PTZ 가능한 첫 번째 프로파일이 자동 선택됩니다.

### Proxy 모드 안내

카메라 추가 모달 하단에 연결 흐름이 표시됩니다:

```
브라우저 → PTZ Proxy (WebSocket) → PTZ 카메라 (TCP)
```

PTZ Proxy가 실행 중이어야 카메라에 연결할 수 있습니다.

## 카메라 연결

1. 카메라 목록에서 카메라를 클릭하여 선택합니다.
2. **Connect** 버튼을 클릭합니다.
3. 연결 상태가 **connected** 로 바뀌면 제어가 가능합니다.

### 연결 실패 시

Proxy 모드에서 연결이 5초 내에 실패하면 **PTZ Proxy 연결 실패** 팝업이 자동으로 표시됩니다.

팝업 내용:

- 이미 PTZ Proxy를 설치했다면 창을 닫고 Proxy를 실행 후 재연결 안내
- PTZ Proxy 다운로드 링크 (Cloud Download 또는 업로드된 실행파일)
- 연결 시도 주소 확인 (토글 버튼)

## PTZ 제어

### Pan / Tilt 방향 제어

8방향 + 정지 버튼으로 구성된 방향 패드입니다.

```
  ↖  ↑  ↗
   ←  ■  →
  ↙  ↓  ↘
```

- 방향 버튼을 **누르고 있는 동안** 해당 방향으로 이동합니다.
- 버튼에서 **손을 떼면** 자동으로 정지합니다.
- 가운데 ■ 버튼은 즉시 정지(Stop)입니다.

### Zoom 제어

| 버튼    | 동작                      |
| ------- | ------------------------- |
| **Out** | 줌 아웃 (축소)            |
| **In**  | 줌 인 (확대, 초록색 강조) |

### Focus 제어

| 버튼     | 동작                         |
| -------- | ---------------------------- |
| **Near** | 가까운 쪽으로 초점           |
| **Far**  | 먼 쪽으로 초점 (보라색 강조) |

### 속도 조절

3개의 슬라이더가 제공됩니다.

| 슬라이더       | 범위   | 기본값 |
| -------------- | ------ | ------ |
| Pan/Tilt Speed | 1–100% | 50%    |
| Zoom Speed     | 1–100% | 50%    |
| Focus Speed    | 1–100% | 50%    |

## 프리셋

### 빠른 프리셋 (1–6)

버튼 1~6을 클릭하면 해당 프리셋 위치로 즉시 이동합니다.

### 커스텀 프리셋

숫자 입력 필드에 프리셋 번호(1–255)를 입력한 후:

| 버튼             | 동작                        | PelcoD    | Ujin      |
| ---------------- | --------------------------- | --------- | --------- |
| **Go** (파랑)    | 해당 프리셋으로 이동        | Cmd2=0x07 | Cmd2=0x07 |
| **Set** (하늘)   | 현재 위치를 프리셋으로 저장 | Cmd2=0x03 | Cmd2=0x03 |
| **Clear** (빨강) | 프리셋 삭제                 | Cmd2=0x05 | 미지원    |

> Ujin 프로토콜은 Clear를 지원하지 않습니다.

## 위치 모니터 및 자동 쿼리

### 현재 위치 표시

Pan, Tilt, Zoom, Focus 4개 값을 읽기 전용으로 표시합니다.

### 수동 위치 설정

입력 필드에 0–65535 범위의 값을 입력하여 카메라 위치를 직접 지정할 수 있습니다.

| 프로토콜   | 제공 버튼                                         |
| ---------- | ------------------------------------------------- |
| **PelcoD** | Set Pan, Set Tilt, Set Zoom, Set Focus, Query All |
| **Ujin**   | GoTo Position, Query Position                     |

### 자동 쿼리 (Auto Query)

카메라 연결 시 **기본 ON**으로 동작하며, 설정한 주기마다 카메라 위치를 자동으로 폴링합니다.

| 설정                | 설명                       |
| ------------------- | -------------------------- |
| Auto Query 체크박스 | 자동 쿼리 ON/OFF           |
| 주기 (ms)           | 100–60000ms (기본 1000ms)  |
| 상태 표시           | "● Polling" (초록, 깜빡임) |

**프로토콜별 쿼리 방식:**

| 프로토콜   | 방식                                                  |
| ---------- | ----------------------------------------------------- |
| **PelcoD** | Pan → Tilt → Zoom → Focus 4축 순차 쿼리 (100ms 간격)  |
| **Ujin**   | 단일 `requestPosition` 명령으로 전체 위치 한번에 수신 |

카메라 연결 해제 시 자동으로 OFF됩니다.

## Hex 모니터

화면 하단에 고정된 패널로, 카메라와 주고받는 프로토콜 패킷을 16진수로 실시간 표시합니다.

### 화면 구성

| 요소             | 설명                      |
| ---------------- | ------------------------- |
| TX 카운트 (초록) | 전송한 패킷 수            |
| RX 카운트 (파랑) | 수신한 패킷 수            |
| Tyche 로고       | dark/light 테마 자동 전환 |

### 로그 형식

```
[12:34:56.789] TX | FF 01 00 04 0B 00 10    pan left
[12:34:56.850] RX | FF 01 00 59 00 3F 99    position
```

각 항목: 타임스탬프, TX/RX 구분, Hex 데이터, 설명

### 컨트롤 버튼

| 버튼               | 기능                                              |
| ------------------ | ------------------------------------------------- |
| ⏸ Pause / ▶ Resume | 로그 업데이트 일시정지/재개 (정지 시 노란색 표시) |
| 📥 Export          | 로그를 텍스트 파일로 내보내기                     |
| 🗑 Clear           | 모든 로그 삭제                                    |
| ▲ / ▼              | 패널 접기/펼치기                                  |

- 최대 500개 로그 유지 (순환 버퍼)
- 드래그 핸들로 패널 높이 조절 가능

> **참고**: ONVIF 카메라는 HTTP 통신이므로 Hex Monitor에 TX가 표시되지 않는 것은 정상입니다.

## 앱 설정

헤더의 ⚙️ 아이콘을 클릭하면 설정 모달이 나타납니다.

| 설정                   | 설명                            | 옵션                        |
| ---------------------- | ------------------------------- | --------------------------- |
| Default Protocol       | 새 카메라 추가 시 기본 프로토콜 | PelcoD, ONVIF, Ujin, Custom |
| Default Operation Mode | 기본 동작 모드                  | Direct, Proxy               |
| Proxy WebSocket Port   | Proxy 기본 포트                 | 9902                        |
| Log Level              | 서버 로그 수준                  | Debug, Info, Warning, Error |
| Theme                  | UI 테마                         | Light, Dark, System         |

테마 변경은 즉시 적용되며, **Save Settings** 를 클릭하여 저장합니다.

## PTZ Proxy

### PTZ Proxy란?

브라우저(또는 Electron 앱)가 Private 네트워크의 PTZ 카메라를 제어할 수 있도록 WebSocket ↔ TCP를 중계하는 서비스입니다.

### 제공 형태

| 형태                   | 설명                                 | 권장    |
| ---------------------- | ------------------------------------ | ------- |
| **ptz-proxy-electron** | GUI + 시스템 트레이 앱 (Windows EXE) | ✅ 권장 |

### 다운로드 방법

1. 카메라 연결 실패 시 자동으로 나타나는 팝업에서 다운로드
2. Cloud Download URL (관리자가 설정한 경우)
3. 관리자가 업로드한 실행 파일 다운로드

### 기본 포트

**9902** (앱 내에서 변경 가능)

### 트레이 아이콘

| 동작               | 결과                             |
| ------------------ | -------------------------------- |
| 창의 ✕ 또는 ⊟      | 트레이로 최소화 (서버 계속 실행) |
| 트레이 아이콘 클릭 | 창 열기                          |
| 트레이 우클릭      | 서버 시작/중지, 종료 메뉴        |
| 트레이 메뉴 "종료" | 서버 중지 후 완전 종료           |

### ONVIF 인증 자동 협상

| 시도 순서 | 인증 방식                  | 대상 카메라            |
| --------- | -------------------------- | ---------------------- |
| 1차       | 인증 없음                  | 인증 불필요 카메라     |
| 2차       | HTTP Digest Auth           | 삼성 iPolis, 한화비전  |
| 3차       | HTTP Basic Auth            | 일부 저가 IP 카메라    |
| 4차       | WS-Security PasswordDigest | Axis, Bosch, Sony      |
| 5차       | WS-Security + Basic Auth   | 일부 하이브리드 카메라 |

### 토큰 인증

| 설정       | 설명                                                   |
| ---------- | ------------------------------------------------------ |
| OFF (기본) | 토큰 없이 누구나 WebSocket 연결 가능                   |
| ON         | 웹앱에서 발급한 JWT 토큰이 있어야 연결 허용 (TTL 60초) |

> Desktop 버전도 PTZ Proxy를 내장하고 있지 않으므로 별도로 실행해야 합니다.

### Proxy 파일 탭

사용자에게 PTZ Proxy를 배포하기 위한 파일 관리입니다.

| 항목               | 설명                                                       |
| ------------------ | ---------------------------------------------------------- |
| Cloud Download URL | 외부 다운로드 링크 설정 (Google Drive, S3 등)              |
| 파일 업로드        | `.exe`, `.zip`, `.msi`, `.dmg`, `.sh`, `.bat`, `.appimage` |
| 업로드 위치        | `public/downloads/` 폴더                                   |

업로드된 파일과 Cloud Download URL은 사용자의 Proxy 연결 실패 팝업에 자동으로 표시됩니다.

### 다운로드 팝업 표시 우선순위

1. **Cloud Download** — URL이 설정된 경우 항상 표시
2. **실행 파일 다운로드** — 업로드된 파일이 있으면 표시
3. **설치파일 준비중** — 업로드 파일이 없으면 표시

## 문제 해결

### 카메라 연결이 안 될 때

1. PTZ Proxy가 실행 중인지 확인 (트레이 아이콘)
2. Proxy WebSocket URL이 정확한지 확인 (`ws://[IP]:9902`)
3. 카메라 IP와 포트가 맞는지 확인
4. 방화벽에서 9902 포트 허용 여부 확인
5. 브라우저 콘솔(F12)에서 에러 메시지 확인

### PTZ가 움직이지 않을 때

1. 연결 상태가 **connected** 인지 확인
2. 장치 주소(Address)가 카메라 설정과 일치하는지 확인 (PelcoD/Ujin)
3. 속도 슬라이더가 0이 아닌지 확인
4. Hex 모니터에서 TX 패킷이 전송되는지 확인

### ONVIF PTZ가 동작하지 않을 때

1. Profile Token을 **비워두면** 자동 조회됩니다 (권장)
2. ptz-proxy 로그에서 `프로파일 토큰:` 출력 확인
3. 인증 방식은 자동으로 협상되므로 별도 설정 불필요

### Hex 모니터 TX가 비어있을 때

ONVIF 카메라는 HTTP 통신이므로 Hex Monitor TX가 비어있는 것은 **정상**입니다.
PelcoD / Ujin은 TCP 소켓이므로 TX/RX가 정상 표시됩니다.

### admin 버튼(🛡️)이 보이지 않을 때

1. 로그아웃 후 재로그인 (JWT 토큰 갱신)
2. Neon 대시보드 SQL Editor에서 직접 확인:
    ```sql
    SELECT email, role FROM "User";
    UPDATE "User" SET role = 'admin' WHERE email = '본인이메일';
    ```

### 비밀번호 재설정 이메일이 오지 않을 때

1. `.env` 파일에 SMTP 설정이 올바른지 확인
2. Gmail 사용 시: Google 계정 → 보안 → 앱 비밀번호 생성 필요
3. 서버 로그에서 SMTP 관련 에러 확인

### HTTPS 환경에서 ws:// 차단

브라우저가 HTTPS 페이지에서 `ws://` 연결을 차단할 수 있습니다.

- nginx 등으로 `wss://` (WebSocket Secure) 프록시 구성
- 또는 브라우저 설정에서 해당 사이트의 혼합 콘텐츠 허용
